Lightweight terminal user interface

GGUI::SYMBOLS

Contains commonly used Unicode characters and other constant used to render the UTF elements to the finalized buffer.

GGUI::TIME

Contains useful time constants to scale with.

GGUI::BUTTON_STATE

Used to store the keyboard states for every individual state.

GGUI::CONSTANTS

Contains constants about ASCII Escape Code Sequences.

GGUI::BUTTON_STATES

Contains all the key values for the KEYBOARD_STATES

GGUI::RGB

The main class to represent colors in abstract format, before it gets transformed into UTF.

GGUI::RGBA

Extends: RGB class, to add Alpha channel.

GGUI::COLOR

Contains pre-made RGB colors.

GGUI::Vector2

Used for precise vector calculation for 2D space.

GGUI::Vector3

Extends Vector2, used for precise vector calculation for 3D space.

GGUI::COORDINATES

The main class to represent the locations for drawn elements. Will migrate to Vector3 in future.

GGUI::UTF_FLAG

Contains bitmasks about UTF being:

  • IS_ASCII
  • IS_UNICODE
  • ENCODE_START
  • ENCODE_END

GGUI::UTF

Container class for abstract representation of the individual CELL to be drawn to the Terminal.

GGUI::Event

Mainly blank class to help OOP structure with Criteria to start the Event and its extending classes.

GGUI::Input

Extends Event class to add the location of the input as mouse input and the data the mouse conveys, also used to conveys data from the virtual KEYBOARD_STATES. The mouse data is passed as a Coordinates object.

GGUI::Action

Extends Event class to add a Job, which is then executed when the Event Criteria is met.

GGUI::MEMORY_FLAGS

Contains bitmasks used by the Memory class to tell if the Memory is Prolonged or Re-Triggered on construct.

GGUI::Memory

Extends Action class to add the start time and the length at which the memory Job is then activated.

GGUI::Margin

Used by the Margin Value to be accessed as the Value member of the VALUE oop object pointer.

GGUI::VALUE_TYPES

Contains the type of which the base class Value is then casted to.

GGUI::Value

The base class to represent the all different value types. Note that all classes which extend this class, will have to have their own member named as Value, and that they also have their own custom override functions for:

  • VALUE* Copy()
  • VALUE* Parse(std::string val)

GGUI::Number_Value

Extends Value class to add the number value.

GGUI::RGB_Value

Extends Value class to add the RGB value.

GGUI::Bool_Value

Extends Value class to add the bool value.

GGUI::Coordinates_Value

Extends Value class to add the Coordinates value.

GGUI::Margin_Value

Extends Value class to add the Margin value.

GGUI::Shadow_Value

Extends Value class to add the Shadow value.

GGUI::STYLES

Contains the string namings for all the different styling methods.

GGUI::STAIN_TYPE

Contains bitmasks representing the different rendering conditions.

GGUI::Stain

Contains the data for what to refresh in the local rendering buffer.

GGUI::Flags

Legacy way to call the Styling names, not used anymore.

GGUI::State

Contains the different states in which an Element can be in:

  • UNKNOWN
  • RENDERED
  • HIDDEN

GGUI::SETTINGS

Contains the preferences for which GGUI adapts.

GGUI::Element

The base class for all the different renderable elements.

GGUI::TEXT_LOCATION

Contains the different locations in which the text can be anchored:

  • Left
  • Center
  • Right

GGUI::Text_Field

Extends the Element class to add the text field. Also has the capability to anchor the text into different Text Locations. Can accept user input and store written data into itself when `Allow_Text_Input` is enabled. Also allows the user to register custom handler function to handle user given text input with: 

                        
    Input(std::function< void(char)>Then)

GGUI::FILE_STREAMER

File handler for HTML reader in GGUI, capable of notifying custom function handler on when the file is changed. 

                        
    void Add_File_Stream_Handle(std::string File_Handle, std::function< void()> Handle)

GGUI::Button

Extends Text Field class to add easy to use On_Click function.

GGUI::Abstract_Frame_Buffer

A vector of UTF, representing the renderable buffer.

GGUI::Frame_Buffer

A string containing the final data which is then written to the terminal file handle.

GGUI::Pause_Render

An Atomic Boolean representing if the rendering pipeline is paused or not.

GGUI::Pause_Event_Thread

An Atomic Boolean representing if the Event pipeline is paused or not.

GGUI::Max_Width

A variable containing the Terminal Width.

GGUI::Max_Height

A variable containing the Terminal Height.

GGUI::Remember

A vector containing the Memory to be run at given time.

GGUI Naming Practices

In GGUI classes or containers like namespaces or enums follow are in caps if the container or class represents a multiple. Examples:

  • COORDINATES
  • VALUE_TYPES
  • Element
Notice how the name 'Element' represents a singular and thus is not all caps, meanwhile the 'COORDINATES' or 'VALUE_TYPES' are in all caps, because they contain or represent multiple. Other cases where the name is in all caps is with abbreviations or any enum class.
All named variables which consist of multiple words are connected with an underscore.

Building For Termux

When building for Termux especially for Android install the following packages:

  • meson
  • g++
  • libandroid-execinfo
After installing the required builder tools, you can build the project by the following: 
                        
    meson setup Build
    meson compile -C Build
If you have previously already builded change the first line into: 
                        
    meson setup --wipe Build
And then add the other lines after it.
