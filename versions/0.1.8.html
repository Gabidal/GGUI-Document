Overview

Major restructuring and modernization of the project. Heaps of optimization and custom containers made for highly optimal results. Whole new styling system for more HTML / XML like styling and node tree construction. Internal components are clipped off from auto generated header exports. Since this update nearly re-wrote the whole project, many commits that look like fixes are about issues brought by the re-write.

API Changes

  • Input events now use IVector2 instead of Coordinate struct.
  • Many (if not all) classes are now written in lower camelCase.
  • Vector2 is now correctly named FVector2 for containing floating point coordinates.
  • GGUI::mouse is now an IVector2 type.
  • You can now give percentage floating point values for almost any value-based styling (width, color channel, etc.). Example:
    • element( width(100) | height(100) | // set the parent element as 100x100 node(new element( width(0.50f) | height(0.50f) // this child element is now 50x50 and updates as its parents dimensions change )) )
  • GGUI no longer has RGBA; opacity is provided as a single floating value per element node.
  • RGB lerp can use Gamma color correct compute (higher cost) or a cheap simple lerp formula.
  • Configure gamma correction via setting or command line argument: enableGammaCorrection
  • GGUI accepts many configurable settings at call time via the args. Hyphens are removed so you can add single hyphen, double hyphen or no hyphens.
  • Node structure follows an HTML / XML like style. GGUI provides on_init() and other on_* state change handlers the user can customize.
  • Because JS and similar ecosystems use underscores for their handlers, GGUI uses the same naming convention: on_*.
  • GGUI::Main is not accessible anymore; it was moved under GGUI::INTERNAL::Main and is intended to be accessed through GGUI::getRoot().
  • Titles are now native to the element class; the window class was removed.
  • Pixel-based canvas removed (only terminal cell based canvas retained). For pixel perfect canvas look up GGDirect (GGUI with DRM backend support).
  • Button class removed; a normal textField with on_click() offers the same behavior.
  • SwitchBoxes now have singleSelect(true) styling which disables other marked SwitchBoxes in the same parent for simple single-choice lists.
  • Shadow styling removed (will be replaced later with full shader-like customizations).

Fixes

  • Fixed transparency not propagating properly to transparent layers of text (was not obtaining color from layers below).

Optimizations

  • Introduced custom string utilities to prevent unnecessary concatenations:
    • compactString: Holds a std::variant of either a char or a Unicode letter.
    • superString: Holds a predetermined list of compactStrings, capable of concatenating smaller superStrings.
    • The entire rendering pipeline is based on these two classes.
  • Many deeply nested getters made constexpr to improve inlining and runtime.
  • Implemented custom fastVector to replace certain std::vector usages for more cache-friendly behavior between render passes.

For Maintainers

  • Added proper scripts (Linux & Windows) for init, build, export, test and benchmarking with various tools.
  • GGUI now accepts -DGGUI_DEBUG, -DGGUI_RELEASE or -DGGUI_PROFILE (managed via Meson, can be set manually).
  • Enabling -DGGUI_DEBUG makes GGUI behave like --verbose in other programs.

Internal

  • Added new multithread safety guard wrapper for memory events.
  • Distributed to multithreading:
    • Rendering pipeline
    • Memory and timed events handling
    • User input handling
  • Introduced internally used reportStacktrace() utilizing symbol table lookup and stack to print helpful stacktraces.
  • Utilizes constexpr operators to default many stylings; currently only works on GCC v13+ (not Clang).
  • The GGUI library now links statically with libc and libc++.

Small Additional Notes

  • GitHub Actions CI workflows.
  • New analytics & benchmarking scripts:
    • ./bin/test.sh ./bin/analytics/benchmark.sh ./bin/analytics/time.sh 5 10 # runs for 5 seconds and 10 seconds and displays growth in opcode count
  • Examples organized into structured sub folders:
    • small — single feature focused examples
    • medium — feature combinatorial examples
    • large — applicative examples
  • Developer and contribution tooling (.github, guidelines).
  • Percentage-based positioning accounts for parent.width/height minus the node's own size.

Example positioning with percentages:

element( width(100) | height(100) | node(new element( width(10) | height(10) | position(STYLES::bottom + STYLES::left) // Forces element to bottom-left, offset by its 10x10 so it stays in-bounds. )) )
